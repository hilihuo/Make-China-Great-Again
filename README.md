# 中华之路：AI 重访历史现场

一个以 26 个历史纪元为主线的沉浸式 Web 3D 人文博物馆。项目把历史人物、关键事件、历史现场、人文图片与可交互展品放在同一条时间长卷中，让用户不只阅读年代与结论，还能从人物处境、器物细节和生活现场理解历史。

核心体验使用 Vite、Three.js、GSAP 和 Web Audio API 构建，无需后端服务。项目既可在浏览器中预览，也已封装为 Windows 安装程序和 Android APK；三种方式使用同一套历史内容、图片与 3D 展品。

## 三种使用方式

| 使用方式 | 适合场景 | 启动方法 | 是否需要开发环境 |
| --- | --- | --- | --- |

| Windows 安装版 | 日常电脑使用、展览演示 | 安装 `China-Humanities-Museum-1.0.0-Setup.exe` 后从开始菜单启动 | 不需要 Node.js 或浏览器开发服务器 |

已经生成的安装包位于本机 `release/`。详细安装方法见 [NATIVE_APPS.md](NATIVE_APPS.md)，当前版本校验值见 [RELEASE_1.0.0.md](RELEASE_1.0.0.md)，通过 GitHub Desktop 提交和发布见 [GITHUB_DESKTOP_UPLOAD.md](GITHUB_DESKTOP_UPLOAD.md)。

## 项目目标

传统时间线容易把历史压缩成朝代名称和事件列表。本项目采用“用 AI 重访一段历史”的设计思路，将每个纪元拆解为五个互相补充的观察入口：

1. **人文图片**：呈现社会生活、文化活动、器物使用和人与环境之间的关系。
2. **人物还原**：结合人物画像与叙事文本，强调具体处境、动作、身份和精神选择。
3. **事件还原**：把抽象历史事件放回人物、制度和现场中理解。
4. **历史现场**：呈现空间、光线、劳动、日常和社会氛围，不采用模板化项目旁白。
5. **3D 展示**：提供可旋转展品、参考实物图、关键细节说明和语音讲解。

视觉内容属于基于史料特征和项目叙事进行的数字化重建，用于人文体验与教学展示，不替代博物馆定名、考古报告或专业历史研究。

### 2. 纪元首页

进入纪元后，页面由四个主要区域构成：

- **左侧人文史卷**：常驻时间线，可快速跳转到任意纪元。
- **左上纪元卡片**：显示当前序号、标题和人文主题。
- **中部人物展示**：自动展开当前纪元代表人物，可拖拽观察主场景。
- **右侧内容卡片**：显示人文介绍、人物与展品数量，以及五个内容入口。

在没有弹窗时，可以使用以下方式切换纪元：

- 鼠标滚轮前后滚动。
- 键盘 `W` / `S` 或方向键。
- 页面底部“上一纪元”“下一纪元”按钮。
- 左侧人文史卷中的纪元按钮。
- 首页左右两侧路线缩略图。

打开人文图片、人物还原、事件还原、历史现场或 3D 展示弹窗后，滚轮只作用于当前内容，不会误切换纪元。

### 3. 人文图像与历史叙事

点击右侧按钮可以打开相应内容：

- **人文图片**：使用左右箭头浏览同一纪元的多张图片，下方显示图片标题和简要说明。
- **人物还原**：以人物图像和文学化历史叙事并置，重点描述人物所处环境、动作、器物和精神状态。
- **事件还原**：从具体参与者和现场关系进入事件，不使用抽象模板化说明。
- **历史现场**：强调空间、声音、照明、劳动和日常物件，使历史成为可感知的场景。

弹窗右上角的高对比关闭按钮可返回纪元首页。关闭后，中部人物展示会恢复并保持显示。

### 4. 3D 展品交互

点击“3D 展示”进入展品弹窗。弹窗左侧为模型或多角度转台，右侧为展品名称、关键细节、历史介绍和语音讲解。

3D 模型操作：

| 操作 | 功能 |
| --- | --- |
| 按住鼠标左键拖动 | 360°旋转模型 |
| 鼠标滚轮 | 拉近或拉远 |
| 按住鼠标右键拖动 | 平移模型 |
| “3D 模型”按钮 | 返回交互模型 |
| “参考图”按钮 | 查看实物或纹理依据 |
| “听讲解”按钮 | 播放或停止中文展品讲解 |

模型根据展品类型采用两种实现方式：

- **程序化完整 3D 几何**：适合陶器、玉器、青铜器、书籍、桌面静物、电子设备等，可从任意角度查看。
- **多视角转台展示**：用于需要高精度人物造像或复杂纹理的展品，以多个一致视角提供旋转观察。

展品背景、灯光和初始缩放会按主体明暗自动调整，保证轮廓与纹理清晰。第 26 纪元的桌面场景包含完整笔记本电脑、陶瓷杯、咖啡液面、动态蒸汽、纸张、圆珠笔、书本和木质桌面。

### 5. 声音系统

点击右上角“音效已关/音效已开”控制背景声场。每个纪元使用不同的基础频率、音阶、呼吸速度和点缀音色，切换纪元时平滑过渡。

声音系统包括：

- 低频环境铺底。
- 钟磬、风声或柔和现代音色。
- 与纪元对应的五声音阶点缀。
- 点击与纪元切换反馈音。
- 3D 展品中文语音讲解。

第 25、26 纪元使用经过低通滤波和动态压缩的双正弦声场，避免尖锐数字脉冲与电流杂音。展品讲解仅在用户点击“听讲解”后播放，不自动朗读。

## 快速开始

### 浏览器预览环境要求

- Node.js 18 或更高版本；如需重新构建 Android 应用，使用 Node.js 22 或更高版本。
- 支持 WebGL 2 和 Web Audio API 的现代浏览器。
- 推荐 Chrome、Edge 或其他 Chromium 浏览器。
- 建议使用独立显卡或开启浏览器硬件加速，以获得更稳定的 3D 帧率。

### 安装依赖

```powershell
cd C:\Users\hilih\Documents\Projects\RW
npm install
```

### 启动开发环境

```powershell
npm run dev -- --host 127.0.0.1
```

浏览器访问：

```text
http://127.0.0.1:3000/
```

Vite 配置已经将开发端口固定为 `3000`。终端需要保持运行；关闭终端或停止进程后，网页将无法访问。

### 构建生产版本

```powershell
npm run build
```

构建结果输出到 `dist/`。本地检查生产包可运行：

```powershell
npm run preview -- --host 127.0.0.1
```

### Windows 安装使用

1. 打开 `release/windows/China-Humanities-Museum-1.0.0-Setup.exe`。
2. 按安装向导完成安装；如果 SmartScreen 提示未知发布者，在确认文件哈希正确后选择继续运行。
3. 从桌面或开始菜单打开“中华人文史卷”。安装后不需要再运行 PowerShell 或访问本地网址。

1. 将 `release/android/China-Humanities-Museum-1.0.0.apk` 传到手机或平板。

3. 安装后打开“中华人文史卷”，建议横屏使用并预留足够存储空间。
4. 如果手机中已有不同签名的旧测试版，应先卸载旧版再安装当前 Release APK。

## 技术设计

### 前端架构

- **Vite**：提供开发服务器、模块加载和生产构建。
- **Three.js**：负责博物馆主场景、灯光、材质、模型、相机和 3D 展品查看器。
- **GSAP**：负责进入旅程、纪元切换和界面过渡。
- **Canvas 2D**：生成代码屏幕、纸张纤维、器物纹样和部分动态纹理。
- **Web Audio API**：实时生成不同纪元的背景音乐，不依赖单一循环音频文件。
- **Web Speech API**：调用浏览器中文语音完成展品讲解。

### 数据驱动

`src/nodes.js` 是纪元内容的主要数据源。每个节点包含：

- 标题、时期、人文介绍和色彩主题。
- 代表人物、身份、引语、人物叙事和朗读文本。
- 展品名称、年代、文字介绍、基础材质与几何信息。
- 粒子类型和背景声参数。

`src/visualSpecs.js` 将节点和展品映射到具体视觉方案。`src/main.js` 负责把数据呈现在界面中，`src/renderer.js` 根据展品类型生成对应模型。

### 3D 材质策略

程序化展品使用 `MeshStandardMaterial` 或 `MeshPhysicalMaterial`，并根据对象类型组合以下信息：

- 金属度、粗糙度、清漆层和透射。
- Canvas 生成的颜色、凹凸与磨损纹理。
- 多层几何表现刻纹、开片、锈迹、纸页和机械结构。
- 主光、补光、轮廓光、环境光和软阴影。
- 按展品亮度设置的对比背景与曝光。

## 内容和素材维护

### 修改纪元文字

编辑 `src/nodes.js` 中对应节点。不要只修改界面中的静态占位文字，因为运行时内容由节点数据覆盖。

### 修改人物、事件或现场图片

图片统一放在 `public/images/`。常见命名包括：

- `character-{节点编号}-{人物编号}.png`
- `event-{节点编号}.png`
- `site-{节点编号}.png`
- 按主题命名的人文图片，例如 `weijin-scholars-direction-fixed.png`

图片映射和多图画廊配置位于 `src/main.js` 的 `getNodeImageGallery()`。

### 修改展品参考图

参考图放在 `public/images/`，图片键由 `src/visualSpecs.js` 的 `artifact.imageKey` 指定。

### 修改或增加 3D 展品

1. 在 `src/visualSpecs.js` 为展品指定唯一 `variant`。
2. 在 `src/renderer.js` 中实现对应的创建函数。
3. 在 `buildProceduralArtifact()` 中注册该 `variant`。
4. 为所有网格设置合理的几何厚度、材质、阴影和初始缩放。
5. 检查正面、背面、左右侧、顶部和底部，避免只为一个视角制作。

### 调整纪元音乐

在 `src/nodes.js` 的 `ambientStyle` 中调整：

```js
ambientStyle: {
  type: 'drone',
  baseFreq: 60,
  chimeFreq: 120,
  lfoFreq: 0.1
}
```

- `type`：声场类型，可使用 `drone`、`wind`、`bell` 或 `sine`。
- `baseFreq`：环境铺底的基础频率。
- `chimeFreq`：点缀音的基础频率。
- `lfoFreq`：音量呼吸变化速度。

最终输出会经过低通滤波和动态压缩。避免设置过高的 `chimeFreq`，也不要使用未经滤波的锯齿波或高增益三角波。

## 常见问题

### `http://127.0.0.1:3000/` 无法打开

确认运行开发服务器的 PowerShell 窗口没有关闭，并重新执行：

```powershell
npm run dev -- --host 127.0.0.1
```

如果端口被占用，先结束旧的 Vite 进程，或临时指定其他端口。

### 页面打开但 3D 区域为空

- 检查浏览器是否启用硬件加速。
- 更新显卡驱动和浏览器。
- 在浏览器开发者工具中查看是否有 WebGL 错误。
- 避免同时打开过多高分辨率 3D 页面。

### “听讲解”没有声音

- 必须由用户点击“听讲解”，浏览器不会自动播放讲解。
- 确认系统音量和当前标签页没有静音。
- 确认系统安装了中文语音；Chrome/Edge 会优先选择可用的普通话语音。
- 关闭弹窗再重新打开，可以重置被中断的朗读队列。

### 背景音乐没有声音

点击右上角“音效已关”按钮，使其显示为“音效已开”。浏览器要求音频必须在用户交互后启动，因此首页加载时默认静音。

### 图片仍显示旧版本

使用 `Ctrl + F5` 强制刷新。对于必须立即替换的关键图片，建议使用新文件名并同步更新 `src/main.js` 中的映射，避免浏览器复用缓存。

### 页面滚动时意外切换纪元

内容弹窗打开时，程序会阻止纪元滚轮导航。如果仍出现问题，确认当前弹窗没有被浏览器扩展或自定义样式意外隐藏。

## 验证建议

提交修改前至少执行：

```powershell
node --check src/main.js
node --check src/renderer.js
node --check src/sound.js
npm run build
```

涉及 3D 展品时，还应人工检查：

1. 默认视角是否完整显示全部展品。
2. 旋转到背面、侧面、顶部和底部后结构是否合理。
3. 材质是否出现漂浮贴片、错误高亮或穿模。
4. 参考图切换、缩放、平移和关闭弹窗是否正常。
5. 桌面端与移动端是否存在文字截断或按钮重叠。

## 项目定位

“中华之路”不是把 AI 图片简单排列成历史图册，而是尝试把图像、人物叙事、事件解释、空间现场、器物细节和声音组织成一条可探索的人文史卷。用户走过的并非只有朝代顺序，而是人在技术、制度、战争、迁徙、信仰和日常生活中的选择。
